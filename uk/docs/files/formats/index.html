Векторні формати​

SVG (Scalable Vector Graphics)​

Розширення: .svg MIME-тип: image/svg+xml Імпорт: Прямий аналіз векторів або трасування растру Експорт: Експорт об'єктів (тільки геометрія)

Що таке SVG?

SVG — це формат векторних зображень на основі XML. Це переважний формат для імпорту дизайну в Rayforge.

Підтримувані функції:

  • ✓ Шляхи (лінії, криві, дуги)
  • ✓ Базові фігури (прямокутники, кола, еліпси, багатокутники)
  • ✓ Групи та перетворення
  • ✓ Кольори обведення та заливки
  • ✓ Кілька шарів
  • ✓ Перетворення координат (перенесення, обертання, масштабування)

Непідтримувані/обмежені функції:

  • ✗ Текст (необхідно спочатку перетворити на шляхи)
  • ✗ Градієнти (спрощуються або ігноруються)
  • ✗ Фільтри та ефекти (ігноруються)
  • ✗ Маски та шляхи обрізки (можуть працювати некоректно)
  • ✗ Вбудовані растрові зображення (імпортуються окремо, якщо можливо)
  • ✗ Складні стилі обведення (пунктир можуть бути спрощені)
  • ✗ Символи та елементи use (екземпляри можуть не оновлюватися)

Примітки до експорту:

При експорті деталі в SVG Rayforge експортує геометрію як векторні шляхи з:

  • Тільки обведенням (без заливки)
  • Одиницями в міліметрах
  • Чорним кольором обведення

Найкращі практики:

  1. Використовуйте формат Plain SVG (не Inkscape SVG або інші варіанти, специфічні для інструментів)
  2. Перетворіть текст на шляхи перед експортом
  3. Спростіть складні шляхи для зменшення кількості вузлів
  4. Зведіть групи де можливо
  5. Видаліть невикористані елементи (напрямні, сітки, приховані шари)
  6. Встановіть одиниці документа в мм (рідна одиниця Rayforge)

Рекомендації ПЗ:

  • Inkscape (безкоштовний) - Відмінна підтримка SVG, рідний формат

DXF (Drawing Exchange Format)​

Розширення: .dxf MIME-тип: application/dxf, image/vnd.dxf Імпорт: Прямий аналіз векторів Експорт: Експорт об'єктів (тільки геометрія)

Що таке DXF?

DXF — це формат креслень AutoCAD, широко використовуваний для обміну даними CAD.

Підтримувані версії:

  • ✓ R12/LT2 (рекомендовано — найкраща сумісність)
  • ✓ R13, R14
  • ✓ R2000 та пізніші (зазвичай працює, але R12 безпечніше)

Підтримувані сутності:

  • ✓ Лінії (LINE)
  • ✓ Полілінії (LWPOLYLINE, POLYLINE)
  • ✓ Дуги (ARC)
  • ✓ Кола (CIRCLE)
  • ✓ Сплайни (SPLINE) — конвертуються в полілінії
  • ✓ Еліпси (ELLIPSE)
  • ✓ Шари

Непідтримувані/обмежені функції:

  • ✗ 3D-сутності (використовуйте 2D-проекцію)
  • ✗ Розміри та анотації (ігноруються)
  • ✗ Блоки/вставки (можуть не інстанціюватися коректно)
  • ✗ Складні типи ліній (спрощуються до суцільних)
  • ✗ Текст (ігнорується, спочатку перетворіть на контури)
  • ✗ Штрихування (можуть бути спрощені або ігноровані)

Примітки до експорту:

При експорті деталі в DXF Rayforge експортує:

  • Лінії як сутності LWPOLYLINE
  • Дуги як сутності ARC
  • Криві Безьє як сутності SPLINE
  • Одиниці в міліметрах (INSUNITS = 4)

RFS (Ескіз Rayforge)​

Розширення: .rfs MIME-тип: application/x-rayforge-sketch Імпорт: Прямий (деталі на основі ескізів) Експорт: Експорт об'єктів (деталі на основі ескізів)

Що таке RFS?

RFS — це власний параметричний формат ескізів Rayforge. Він зберігає всі геометричні елементи та параметричні обмеження, дозволяючи зберігати та спільно використовувати повністю редаговані ескізи.

Функції:

  • ✓ Усі геометричні елементи (лінії, дуги, кола, прямокутники тощо)
  • ✓ Усі параметричні обмеження
  • ✓ Розмірні значення та вирази
  • ✓ Області заливки

Коли використовувати:

  • Збереження багаторазових параметричних дизайнів
  • Спільне використання редагованих ескізів з іншими користувачами Rayforge
  • Архівування роботи в процесі

PDF (Portable Document Format)​

Розширення: .pdf MIME-тип: application/pdf Імпорт: Прямі вектори (з підтримкою шарів) або рендеринг і трасування Експорт: Не підтримується

Що таке імпорт PDF?

Файли PDF можуть містити реальні векторні шляхи, і Rayforge імпортує їх безпосередньо, коли вони доступні — надаючи вам таку ж чисту геометрію, яку ви б отримали з SVG. Якщо PDF містить шари, кожен шар може бути імпортований як окрема деталь.

Для PDF без придатного векторного вмісту (відскановані документи, фотографії) Rayforge використовує рендеринг та трасування.

Можливості:

  • ✓ Прямий векторний імпорт для векторних PDF
  • ✓ Виявлення та вибір шарів — оберіть шари для імпорту
  • ✓ Резервний рендеринг та трасування для растрового вмісту

Обмеження:

  • Тільки перша сторінка — багатосторінкові PDF імпортують сторінку 1
  • Текст може потребувати перетворення на контури у вихідному додатку

Коли використовувати:

  • PDF, отримані від дизайнерів, що містять векторні ілюстрації
  • Будь-який PDF з добре організованими шарами
  • Коли SVG або DXF недоступні з джерела

Растрові формати​

Усі растрові формати імпортуються шляхом трасування — автоматично конвертуються у векторні шляхи.

PNG (Portable Network Graphics)​

Розширення: .png MIME-тип: image/png Імпорт: Трасування у вектори Експорт: Не підтримується

Характеристики:

  • Стиснення без втрат — без втрати якості
  • Підтримка прозорості — альфа-канал збережено
  • Підходить для: Логотипів, лінійної графіки, знімків екрану, усього, що потребує прозорості

Якість трасування: (Відмінна для висококонтрастних зображень)

Найкращі практики:

  • Використовуйте PNG для логотипів та графіки з чіткими краями
  • Забезпечте високий контраст між переднім планом та фоном
  • Прозорий фон працює краще, ніж білий

JPEG (Joint Photographic Experts Group)​

Розширення: .jpg, .jpeg MIME-тип: image/jpeg Імпорт: Трасування у вектори Експорт: Не підтримується

Характеристики:

  • Стиснення з втратами — певна втрата якості
  • Без прозорості — завжди має фон
  • Підходить для: Фотографій, зображень з безперервними тонами

Якість трасування: (Добра для фотографій, але складно)

Найкращі практики:

  • Використовуйте високоякісний JPEG (низьке стиснення)
  • Збільште контраст перед імпортом
  • Розгляньте попередню обробку в редакторі зображень
  • Краще спочатку перетворити на PNG, якщо можливо

BMP (Растрове зображення)​

Розширення: .bmp MIME-тип: image/bmp Імпорт: Трасування у вектори Експорт: Не підтримується

Характеристики:

  • Без стиснення — великі розміри файлів
  • Простий формат — широко сумісний
  • Підходить для: Простої графіки, виведення старого ПЗ

Якість трасування: (Добра, але не краща за PNG)

Найкращі практики:

  • Перетворіть на PNG для меншого розміру файлу (без втрати якості)
  • Використовуйте лише якщо вихідне ПЗ не може експортувати PNG/SVG
